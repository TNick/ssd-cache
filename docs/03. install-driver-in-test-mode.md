# Install Driver in Test Mode

Run these from an elevated PowerShell in the repo root.

```powershell
.\scripts\build.ps1 -Preset windows-driver-debug
```

```powershell
$pkg = Resolve-Path .\build\driver\Debug
$signtool = 'C:\Program Files (x86)\Windows Kits\10\bin\10.0.26100.0\x64\signtool.exe'
$inf2cat = 'C:\Program Files (x86)\Windows Kits\10\bin\10.0.26100.0\x86\Inf2Cat.exe'
```

```powershell
$cert = New-SelfSignedCertificate `
	-Type CodeSigningCert `
	-Subject 'CN=ssd-cache local test driver signing' `
	-CertStoreLocation Cert:\LocalMachine\My `
	-KeyExportPolicy Exportable `
	-KeyAlgorithm RSA `
	-KeyLength 3072 `
	-HashAlgorithm SHA256 `
	-NotAfter (Get-Date).AddYears(3)
```

```powershell
Export-Certificate -Cert $cert -FilePath "$pkg\ssd-cache-test.cer"
```

```powershell
Import-Certificate -FilePath "$pkg\ssd-cache-test.cer" `
	-CertStoreLocation Cert:\LocalMachine\Root
Import-Certificate -FilePath "$pkg\ssd-cache-test.cer" `
	-CertStoreLocation Cert:\LocalMachine\TrustedPublisher
```

```powershell
& $inf2cat /driver:$pkg /os:10_GE_X64 /verbose
```

```powershell
& $signtool sign /v /fd SHA256 /sm /s My /sha1 $cert.Thumbprint `
	/tr http://timestamp.digicert.com /td SHA256 "$pkg\cachemon.cat"
& $signtool sign /v /fd SHA256 /sm /s My /sha1 $cert.Thumbprint `
	/tr http://timestamp.digicert.com /td SHA256 "$pkg\cachemon.sys"
```

```powershell
& $signtool verify /v /pa /c "$pkg\cachemon.cat" "$pkg\cachemon.inf"
& $signtool verify /v /pa "$pkg\cachemon.sys"
```

Use `/pa` for the local self-signed test certificate. `/kp` validates
kernel-mode/release policy and can fail with a Microsoft-root chain error for
this local test certificate.

```powershell
bcdedit.exe /set TESTSIGNING ON
```

Reboot after TESTSIGNING ON.

If bcdedit says the value is protected by Secure Boot policy, suspend BitLocker
first if enabled, reboot into firmware, disable Secure Boot, then run the
bcdedit command again.

After reboot, install/load:

```powershell
pnputil.exe /add-driver "$pkg\cachemon.inf" /install

# Microsoft PnP Utility
```

```powershell
fltmc.exe load CacheMon

# No output expected.
```

```powershell
fltmc.exe filters | findstr /i CacheMon

# CacheMon                                6       370030         0
```

To undo test mode later:

```powershell
bcdedit.exe /set TESTSIGNING OFF
```

Microsoft references: test-signing requires TESTSIGNING and reboot, Secure Boot
can block changing it, and driver packages are normally signed through a
catalog generated by Inf2Cat and signed with SignTool:

- https://learn.microsoft.com/en-us/windows-hardware/drivers/install/the-testsigning-boot-configuration-option,
- https://learn.microsoft.com/en-us/windows-hardware/drivers/install/test-signing,
- https://learn.microsoft.com/en-us/windows-hardware/drivers/devtest/signtool.
